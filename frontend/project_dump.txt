## 🔐 Enabling Password Login – Step-by-Step Guideline

This guide explains how to add **password-based login** alongside the existing **OTP** and **Face ID** methods.  
You will modify **backend endpoints**, **frontend API clients**, **authentication context**, and the **login UI** to support phone/email + password authentication.

---

### 📁 1. Backend (Django + Django Ninja)

#### a) `apps/accounts/api.py` – add new login endpoint

**Location:** `backend/apps/accounts/api.py`

Add a new route **`/login/`** that accepts `phone_number` (or `email`) and `password`, validates credentials, and returns JWT tokens.

```python
# Inside router, after the existing OTP endpoints

@router.post("/login/", auth=None)
def password_login(request, data: dict):
    """
    Login with phone_number and password.
    Expected payload: {"phone_number": "0712345678", "password": "your_password"}
    """
    phone = data.get("phone_number")
    password = data.get("password")

    if not phone or not password:
        return {"error": "Phone number and password are required"}, 400

    # Django's authenticate uses USERNAME_FIELD (phone_number)
    user = authenticate(request, phone_number=phone, password=password)

    if user is None:
        return {"error": "Invalid credentials"}, 401

    if not user.is_active:
        return {"error": "Account is deactivated"}, 403

    # Generate tokens
    tokens = create_token_pair(user)
    AccountService.increment_login_count(user)

    # Return same structure as OTP verify
    return {
        "access": tokens["access"],
        "refresh": tokens["refresh"],
        "user": AccountService.serialize_user(user),
    }
```

**Why:** This endpoint reuses the same token generation and user serialization logic as OTP login, ensuring a consistent frontend experience.

---

#### b) `apps/accounts/schema.py` – define a schema for password login

Add a Pydantic schema for request validation (optional but recommended).

```python
# Inside schema.py
class PasswordLoginIn(Schema):
    phone_number: str
    password: str
```

Then in `api.py` use `data: PasswordLoginIn` instead of a raw dict.

**Why:** Better validation and auto-documentation.

---

#### c) Ensure password hashing is active

Django’s `AbstractBaseUser` already uses `set_password()` which hashes with **PBKDF2** by default.  
No changes are needed; it is **encrypted** and secure.

**Why:** Exposes the new endpoint to the frontend.

#### b) `frontend/src/contexts/AuthContext.jsx` – add password login action

Add a new `loginWithPassword` function to the AuthContext.

```javascript
const loginWithPassword = useCallback(async (phone, password) => {
  setLoginLoading(true);
  setError(null);
  try {
    const response = await apiClient.post('/accounts/login/', {
      phone_number: phone,
      password: password,
    });
    const { access, refresh, user } = response.data;
    storeTokens(access, refresh);
    storeUser(user);
    setUser(user);
    return user;
  } catch (err) {
    const message = err.response?.data?.error || 'Login failed.';
    setError(message);
    throw new Error(message);
  } finally {
    setLoginLoading(false);
  }
}, []);
```

Add this to the context value object.

**Why:** Centralizes authentication logic and manages loading/error states.

const handlePasswordLogin = async (e) => {

    return;

### 📁 5. Testing & Security Notes

- **Encryption:** Django uses PBKDF2 with SHA-256 (default). Passwords are never stored in plain text.
- **Force users to set a password:** You may need to add a `password` field in signup or prompt users to set one later (e.g., after first OTP login, show a "Set Password" option).
- **Existing users** who never set a password will not be able to log in with password – they must first reset their password (via `/forgot-password`).
- **Rate limiting:** Consider adding throttling to the login endpoint to prevent brute-force attacks.
- **2FA:** If 2FA is enabled, the password login should also require 2FA. You can integrate it similarly to OTP flow (check `two_factor_enabled` and return `require_2fa` flag).
