# Academe API Documentation

**Base URL**: `http://localhost:8000/`  
**Authentication**: JWT Bearer token (except for public endpoints like signup/OTP)  
**Content-Type**: `application/json` (unless multipart/form-data for file uploads)

---

## 1. Authentication

### 1.1 Signup
`POST /accounts/signup/`

**Request body**
```json
{
  "phone_number": "0712345678",
  "admission_number": "ADM123",
  "full_name": "John Doe",
  "email": "john@example.com",
  "class_name": "CS 3A",
  "institution": "JKUAT"
}
```

**Response**
```json
{"message": "Registration successful"}
```

---

### 1.2 Request OTP
`POST /accounts/request-otp/`

**Request**
```json
{"phone_number": "0712345678"}
```

**Response**
```json
{
  "otp": "123456",
  "message": "OTP generated successfully"
}
```
> In production, the OTP is sent via SMS and not returned in the response.

---

### 1.3 Verify OTP & Login
`POST /accounts/verify-otp/`

**Request**
```json
{
  "phone_number": "0712345678",
  "otp": "123456"
}
```

**Response**
```json
{
  "access": "<JWT access token>",
  "refresh": "<JWT refresh token>",
  "user": {
    "id": "uuid",
    "full_name": "John Doe",
    "phone_number": "0712345678",
    "role": "student",
    ...
  }
}
```

---

### 1.4 Refresh Token
`POST /accounts/refresh-token/`

**Request**
```json
{"refresh": "<refresh token>"}
```

**Response**
```json
{
  "access": "<new access token>",
  "refresh": "<new refresh token>"
}
```

---

### 6.4 Claim an Item
`POST /found-items/items/{item_id}/claim/`  
**Auth**: required  
Starts or resumes a claim process.

**Response**
```json
{
  "claim_id": "uuid",
  "status": "pending",
  "next_step": "security_question"
}
```

---

### 6.5 Claim Flow Endpoints (all auth required)
- **Verify Ownership** `POST /found-items/items/{item_id}/verify-ownership/`  
- **Answer Security Question** `POST /found-items/claims/{claim_id}/answer-security/`  
- **Submit Evidence** `POST /found-items/claims/{claim_id}/submit-evidence/` (multipart)  
- **Initiate Payment** `POST /found-items/claims/{claim_id}/initiate-payment/`  
- **Check Payment Status** `GET /found-items/claims/{claim_id}/payment-status/`  
- **Confirm Receipt** `POST /found-items/claims/{claim_id}/confirm-receipt/`  

---

### 6.6 Admin Endpoints (for found items)

## General Notes

- All endpoints return JSON.  
- Pagination is not implemented by default (most endpoints return all results), but can be added via `?limit=&offset=` on some.  
- For file uploads (found items, profile pic, evidence), use `multipart/form-data`.  
- The base URL for development is `http://localhost:8000/`; the API root is the same (no `/api` prefix).  
- Use the `Authorization: Bearer <access_token>` header for authenticated requests.  
- Refresh tokens are obtained at login and can be used at `/accounts/refresh-token/` to get a new access token without re-authentication.
